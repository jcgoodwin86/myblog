  Project Review: jcgoodwin86/myblog

  ---
  Templ Components

  Structure — good. The three-tier separation (layout → pages → components) is clean and idiomatic. Data flows correctly from handlers into
  components with no business logic leaking into templates.

  Bug: hardcoded placeholder text in postcard.templ.
  The card renders a literal string instead of post.Description:

  <p>"You can't have your cake and eat it too"... This one needs to be buried in a shallow grave.</p>

  Should be { post.Description } (guarded for empty string, since description is optional).

  Bug: unhandled render error in post.go.
  home.go correctly checks the error from .Render(). post.go does not:

  // post.go:40 — return value ignored
  pages.Post(*postData).Render(r.Context(), w)

  href without templ.SafeURL(). In postcard.templ the slug is concatenated directly into an href attribute value. Templ auto-escapes attribute
  content, so this is not a direct XSS risk, but the idiomatic Templ approach is templ.SafeURL("/posts/" + post.Slug) to make intent explicit.

  Missing Content-Type header in post.go. home.go sets w.Header().Set("Content-Type", "text/html") but post.go does not. Go's http.ResponseWriter
  will sniff and set it automatically, but it is better to be explicit.

  Dead nav links. nav.templ links to /about and /contact, but no handlers exist for these routes. Clicking them returns a 404 with no custom error
  page.

  ---
  Goldmark Pipeline

  Centralised — good. A single var Markdown in internal/render/markdown.go is the right place. Nothing is scattered.

  Extensions are sensible. meta, Table, Footnote, Strikethrough, and Chroma highlighting all make sense for a blog.

  WithClasses(false) embeds inline styles. This means syntax-highlighted blocks carry inline style= attributes rather than CSS class names. It works,
   but produces larger HTML and makes theming impossible without re-generating content. WithClasses(true) paired with a Chroma stylesheet is the more
   maintainable choice.

  The commented-out style line.

  // highlighting.WithStyle("catppuccin-latte"),

  Without this, Goldmark falls back to the Chroma default (monokai). This is harmless but suggests the intended style was never finalised.

  Parsing is cached — good. LoadAllPosts() at startup preloads everything. LoadPost() also checks the cache before touching disk. Markdown is never
  re-parsed on hot requests. This is the right design for a blog.

  XSS: safe. Goldmark output is assigned to template.HTML, which signals to Go's html/template and Templ that it is pre-trusted. templ.Raw() in
  markdown.templ is the correct unwrapping call here. The chain is sound.

  ---
  net/http → Templ → Goldmark Data Flow

  Handlers are thin — good. Both handlers do only: read from cache → call render → return. No business logic.

  fmt.Println in post.go should be log.Println.

  // post.go:27
  fmt.Println(err)  // inconsistent with the rest of the app which uses log.*

  post_location should be a constant, not a variable. It is never reassigned:

  var post_location string = "content/posts"  // should be: const postLocation = "content/posts"

  Also uses snake_case; Go convention is camelCase for unexported identifiers.

  author are required — grammatical error in postFromMeta:

  return nil, errors.New("author are required")  // should be "author is required"

  ---
  Markdown Content Files

  Frontmatter is consistent. Both real posts use the same schema: title, date, tags, author, with optional description. The parser enforces all
  required fields, which is good.

  description is optional but never shown. The model parses and stores it, but postcard.templ ignores it (see hardcoded text bug above). The post
  detail page (posts.templ) also does not render description in a <meta> tag or as a subtitle.

  date field uses bare YAML date (2026-02-28, no quotes). goldmark-meta uses gopkg.in/yaml.v2, which parses a bare date as a time.Time, not a string.
   The postFromMeta function does meta["date"].(string), which will fail with a type assertion panic (or silent miss) for bare YAML dates. Quoting
  the date ("2026-02-28") in frontmatter is the safe workaround, but this is a latent bug.

  Content directory is flat. content/posts/ works now. If the post count grows, consider content/posts/YYYY/ subdirectories, though that is not
  urgent.

  ---
  Dependencies & go.mod

  go 1.25.0 does not exist. The latest stable release is Go 1.22/1.23. This is likely a typo (1.22.0 or 1.23.0). The toolchain will accept it, but it
   is misleading.

  goldmark-highlighting/v2 is pinned to a pre-release commit hash (v2.0.0-20230729083705-37449abec8cc). There is no stable tagged release, so this is
   unavoidable, but worth noting.

  README says HTMX is in the stack. It is not in go.mod, go.sum, or any template file. The README is out of date.
